### Step 1: 環境セットアップと現状確認

### Step 2: OCR 環境のセットアップ

Codex が使える OCR 手段を確認:

**選択肢A: Tesseract（推奨）**
```bash
# Windows の場合、scoop/choco で導入可能か確認
where tesseract
# 未導入なら scoop install tesseract or choco install tesseract
# 日本語データ tesseract-ocr-jpn / jpn_vert も入れる
```

**選択肢B: Python easyocr**
```bash
pip install easyocr
# 初回実行時に学習モデルがダウンロードされる（〜500MB）
```

**選択肢C: paddleocr**
```bash
pip install paddleocr paddlepaddle
```

**選択肢D: Windows 標準の Snipping Tool / PowerToys Text Extractor**
- GUI操作になるため Codex のスクリプト化に不向き
- 最終手段として手動 OCR

A〜C のいずれかが使えれば自動化可能。**使えるものを選んで進める**。

### Step 3: PDF をページ毎の画像に変換

`pdftoppm`（poppler-utils）が使える環境であれば:
```bash
mkdir -p tmp/r6a_pages
pdftoppm -r 300 -png /path/to/2024r06a_pm_am2_qs.pdf tmp/r6a_pages/page
ls tmp/r6a_pages/
```

または Python:
```bash
pip install pdf2image
python -c "
from pdf2image import convert_from_path
images = convert_from_path('2024r06a_pm_am2_qs.pdf', dpi=300)
for i, img in enumerate(images, 1):
    img.save(f'tmp/r6a_pages/page-{i:02d}.png')
"
```

### Step 4: OCR 実行 + テキスト抽出

例（easyocr 使用）:
```python
import easyocr
reader = easyocr.Reader(['ja', 'en'], gpu=False)
results = reader.readtext('tmp/r6a_pages/page-03.png', detail=0, paragraph=True)
print('\n'.join(results))
```

各ページの問題文・選択肢を抽出し、テキストファイルに保存。

### Step 5: 構造化データへの整形

OCR 結果を `OfficialMorningQuestion` 型に整形:

```ts
{
  id: 'om-R6-1',
  year: 'R6',
  yearLabel: '令和6（2024）',
  number: 1,
  questionText: '<IPA公式の問題文を一字一句正確に>',
  choices: [
    'ア の選択肢の本文',  // 「ア」プレフィックスは含めない
    'イ の選択肢の本文',
    'ウ の選択肢の本文',
    'エ の選択肢の本文',
  ],
  correctIndex: 1,  // 解答例より（イ=1）
  explanation: '',  // Claude が後工程で追記
  categoryId: 'stakeholder',  // 関連カテゴリ、判断つかなければ undefined
  sourceUrl: 'https://www.ipa.go.jp/shiken/mondai-kaiotu/m42obm000000afqx-att/2024r06a_pm_am2_qs.pdf',
}
```

#### categoryId 推定ガイド
問題文の論点で以下から選ぶ。判断つかなければ省略可:
- `stakeholder` — ステークホルダー識別/分析/エンゲージメント
- `team` — チーム/リーダーシップ/動機付け
- `development-approach` — 予測型/適応型/ハイブリッド
- `planning` — スコープ/スケジュール/コスト/EVM/リソース計画
- `project-work` — プロジェクト遂行/コミュニケーション/調達
- `delivery` — 価値実現/受入/品質
- `measurement` — EVM/KPI/メトリクス
- `uncertainty` — リスク管理
- `integration` — 統合/変更管理
- `governance` — ガバナンス/原則/原理
- `tailoring-models` — テーラリング/モデル/方法
- `service-management` — サービスマネジメント（ITIL等）

#### 重要: 一字一句引用ルール
- **OCR の誤認識は必ず元 PDF と目視照合して修正**
- 全角/半角の英数字、長音記号「ー」「－」、丸数字、ローマ数字（Ⅰ Ⅱ）などは原文に合わせる
- 漢字の旧字/新字、送り仮名揺れも原文通り
- 図表が含まれる問題（問題文中に「次の図」「次の表」等）の場合は、本文に「[次の図参照（IPA原PDFを参照）]」と注記し `categoryId` 末尾に `(figure)` を付けておく（Claudeが後で対応判断）
