## 5. 作業手順

### Step 1: 環境セットアップと現状確認
```bash
git pull origin main
git log --oneline -5
npm install
npm run validate-data
npm run build
```

### Step 2: 件数・採番チェック
- [ ] 設問数が **ちょうど 25 件**
- [ ] `id` が `om-R6-1` 〜 `om-R6-25` で連続、重複なし
- [ ] 全 `id` が `^om-R6-\d+$` パターンに準拠

### Step 3: 型整合性チェック（OfficialMorningQuestion 型）
- [ ] `year === 'R6'` 全件
- [ ] `yearLabel === '令和6（2024）'` 全件
- [ ] `number` が 1〜25 連続
- [ ] `questionText` が空でない、`{{blank}}` を含まない（公式午前IIは4択固定）
- [ ] `choices` は **長さ 4 のタプル**、各要素は空でない
- [ ] `correctIndex` は `0|1|2|3` のいずれか
- [ ] `categoryId` が PM 12カテゴリ ID のいずれか（または undefined）
  - 12カテゴリ: stakeholder / team / development-approach / planning / project-work / delivery / measurement / uncertainty / integration / governance / tailoring-models / service-management
  - `(figure)` suffix が **残っていない** こと
- [ ] `sourceUrl` が IPA 問題冊子 PDF URL に統一されている
- [ ] `explanation` が **全件非空**（25問とも空文字でない）

### Step 4: 正解インデックスの照合
解答例 PDF（または以下の正解一覧）と全件突合:
```
問1=イ(1) 問2=ア(0) 問3=ウ(2) 問4=ウ(2) 問5=エ(3)
問6=イ(1) 問7=エ(3) 問8=ウ(2) 問9=ア(0) 問10=エ(3)
問11=イ(1) 問12=イ(1) 問13=イ(1) 問14=ア(0) 問15=イ(1)
問16=イ(1) 問17=エ(3) 問18=イ(1) 問19=ア(0) 問20=エ(3)
問21=ウ(2) 問22=イ(1) 問23=エ(3) 問24=エ(3) 問25=ウ(2)
```
- [ ] 25 問全件で `correctIndex` が解答例と一致

### Step 5: 解説の対応関係チェック
- [ ] 各問の `explanation` で **`correctIndex` の選択肢を支持する説明**になっているか
  - 例: `correctIndex: 1` なら「イ が正解」を裏付ける内容か
- [ ] 他選択肢への言及があれば、それらが誤答理由として妥当か
- [ ] 解説が PMBOK第7版・IPAシラバス・JIS Q 21500 等の権威あるソースに基づいているか
- [ ] 解説中に IPA 原文の改変や引用がないか（独自表現のみであるべき）

### Step 6: IPA 原文の verbatim 検証（サンプリング 5問）
- 問1、問7、問13、問18、問25 の **5問** について、IPA 原 PDF と `questionText`・`choices` を目視で文字単位照合
- 句読点（，。）、全角/半角記号、ローマ数字（Ⅰ Ⅱ）、長音記号、漢字の使用が原本と完全一致しているか確認
- 図表入り問題（問4・問5・問6・問12）は `figure` フィールド構造化済み（§6c参照）

### Step 6b: 図表データ（figure）の妥当性チェック ★追記
- [ ] 問4・問5・問6 は `figure: { type: 'svg', ... }`、問12 は `figure: { type: 'table', ... }` を持つ
- [ ] 各 SVG figure の `viewBox` が `'0 0 W H'` 形式で妥当
- [ ] 各 SVG figure の `content` 内に `<defs>` の `<style>text { paint-order: stroke fill; stroke: white; ... }</style>` が含まれている
  - これは `docs/morning_question_authoring_rules.md` §5.2 で必須化された白ハロー指定
- [ ] table figure の `headers.length` と各 `rows[i].length` が一致
- [ ] `questionText` から旧来の `[図: ...]` テキストが削除されている（SVG/table に代替済み）
- [ ] **SVG 図のセルフチェック（重要）**:
  ```bash
  npx vite-node scripts/render-morning-figures.ts
  for f in tmp/figures/*.svg; do
    name=$(basename "$f" .svg)
    inkscape "$f" --export-type=png --export-filename="tmp/figures/${name}.png" --export-width=800
  done
  ```
  生成された tmp/figures/*.png を目視確認し、レビュー記録に「PNG確認: ○ / 問題: なし」等を明記

### Step 6c: 数式マークアップの妥当性 ★追記
- [ ] 問9 の `questionText` / `explanation` に `^{0.98}` のような上付きマークアップが含まれる
- [ ] 問10 の `choices` 4つのうち、イ・ウ・エ に `frac{...}{...}` 形式のマークアップが含まれる
  - ア は `'X＋Y＋Z'`（分数なし、変更不要）
  - エ は `frac{1}{frac{1}{X}＋frac{1}{Y}＋frac{1}{Z}}` のネスト構造
- [ ] マークアップの波括弧 `{` `}` のバランスが取れている（開閉数一致）
- [ ] 全角イコールや全角波括弧の混入なし

### Step 6d: カテゴリ分布の妥当性 ★追記
- [ ] 25 問の `categoryId` が PM 12カテゴリ ID のみ（`(figure)` 等の suffix 残留なし）
- [ ] 期待される分布:
  - service-management: 5, measurement: 4, planning: 3, development-approach: 3, project-work: 3
  - team: 2, governance: 2, uncertainty: 1, integration: 1, tailoring-models: 1
  - stakeholder: 0, delivery: 0
- [ ] 個別の特殊調整: Q13=tailoring-models（GoF）、Q21=project-work（AI契約）

### Step 7: ファイル冒頭コメントの妥当性
- [ ] コメントに「Codex OCR + Claude 独自解説」の役割分担が明記されている
- [ ] IPA 著作権規約の遵守宣言が含まれている
- [ ] 図表入り問題の扱いが説明されている（SVG・table の構造化保存）
