# ITサービスマネージャ寄り道モード 要件定義 v1.0

作成日: 2026-06-01  
対象アプリ: PM Learning App  
対象範囲: PMアプリ内に同居するITサービスマネージャ試験対策モード

## 1. 背景

PM Learning Appの中に、寄り道としてITサービスマネージャ試験（SM）の合格に必要な知識と演習を詰め込む。
新しいアプリや別デプロイは作らず、既存PMアプリに組み込む。

ただし、既存PMアプリの学習体験、進捗、検索、履歴、バッジ、同期、既存設計ドキュメントには影響を出さない。
トップページの学習メニューにカードを1枚追加することだけを、既存PM側への例外的な変更として許容する。

## 2. 目的

1. ITサービスマネージャ試験の `午前Ⅱ`、`午後Ⅰ`、`午後Ⅱ` に短時間で合格へ近づくための学習モードを提供する。
2. 直近10回分の出題傾向を分析し、頻出テーマを分かりやすく提示する。
3. 初回リリースでは直近1年分の過去問を深く作り込み、利用者確認後に5年分へ拡張する。
4. 午後Ⅱは骨子、評価観点、インフラエンジニア案件の参考答案を備える。
5. PM本編とITSM寄り道モードを機能的・データ的に隔離する。

## 3. 試験情報

アプリ内表記は馴染みのある旧称を主とする。
2026年度以降のCBT名称は補足表示に留める。

| 区分 | アプリ表記 | CBT移行後の補足 | 形式 | 時間 | 合格目標 |
|---|---|---|---|---:|---|
| 午前Ⅱ | 午前Ⅱ | 科目A-2 | 多肢選択式 | 40分 | 60点以上、アプリ推奨80% |
| 午後Ⅰ | 午後Ⅰ | 科目B-1 | 記述式 | 90分 | 60点以上 |
| 午後Ⅱ | 午後Ⅱ | 科目B-2 | 論述式 | 120分 | ランクA相当 |

参照元:
- IPA ITサービスマネージャ試験: https://www.ipa.go.jp/shiken/kubun/sm.html
- IPA CBT方式移行案内: https://www.ipa.go.jp/shiken/2026/ap_koudo_sc-cbt.html
- IPA 令和7年度過去問題: https://www.ipa.go.jp/shiken/mondai-kaiotu/2025r07.html
- IPA シラバス/試験要綱: https://www.ipa.go.jp/shiken/syllabus/gaiyou.html

## 4. スコープ

### 4.1 初回リリース範囲

初回対象は令和7年度春期ITサービスマネージャ試験とする。

- 午前Ⅱ 25問
- 午後Ⅰ 3問
- 午後Ⅱ 2問
- 頻出テーマ分析表示
- 知識ノート
- ITSM専用履歴
- 午後Ⅱ参考答案 各テーマ2本

### 4.2 調査範囲

コンテンツ作成前に、直近10回分のSM過去問、解答例、採点講評、出題趣旨を調査する。
ただし、初回アプリ内過去問データとして投入するのは直近1年分に限定する。

10回分調査は、次のために使う。

- 頻出テーマランキング
- 知識ノートの厚み付け
- 午後Ⅰ解説の観点
- 午後Ⅱ骨子・参考答案の事例設計
- 5年分拡張時の優先順位

### 4.3 初回リリース外

- 午前Ⅰ対策
- AIによる論述自動採点
- 既存PM検索への統合
- 既存PM履歴への統合
- 既存PMバッジ/XPへの統合
- 既存PM同期への統合
- 既存PMドキュメントの改変
- 新規アプリ化、別デプロイ
- 学習時間計測
- ITSM重要マーク

## 5. 既存PMアプリとの隔離要件

ITSMモードはPMアプリ内に同居するが、既存PM本編には混ぜない。

### 5.1 変更してよい既存箇所

- `src/pages/Home.tsx`
  - 学習メニューにITサービスマネージャカードを1枚追加する。
- `src/App.tsx`
  - ITSM専用ルートを追加する。

### 5.2 変更しない既存箇所

- 既存PMドキュメント
- 既存PMカテゴリ
- 既存PM問題データ
- 既存PMノート
- 既存PM検索
- 既存PM履歴
- 既存PMバッジ
- 既存PM XP/レベル
- 既存PM同期
- 既存PM重要マーク

### 5.3 禁止事項

- `pmap:morning:*` にITSM記録を書かない。
- `pmap:tracker:*` にITSM午後Ⅰ記録を書かない。
- `pmap:essay:*` にITSM午後Ⅱ記録を書かない。
- `pmap:important_questions` をITSMで使わない。
- `pmap:gamification` にITSM結果を反映しない。
- ITSMデータを既存検索対象に入れない。
- ITSMイベントを既存活動履歴に入れない。

### 5.4 保存キー方針

ITSMの保存キーはすべて `pmap:sm:*` とする。

| キー | 用途 |
|---|---|
| `pmap:sm:morning:records` | 午前Ⅱ解答履歴 |
| `pmap:sm:afternoon:records` | 午後Ⅰ自己採点履歴 |
| `pmap:sm:essay:attempts` | 午後Ⅱ練習履歴 |
| `pmap:sm:essay:drafts` | 午後Ⅱ下書き |
| `pmap:sm:events` | ITSM専用履歴 |

## 6. 学習方針

50時間という学習時間は記録対象ではなく、学習設計上の目安とする。
アプリ内では、時間計測ではなく試験区分別の達成状況を主表示する。

主役:

- 午前Ⅱ: 正答率、演習済み数、直近不正解
- 午後Ⅰ: 問題別の得点、演習回数
- 午後Ⅱ: 骨子作成、論述練習、自己評価
- 頻出テーマ: S/A/Bランク、午前Ⅱ/午後Ⅰ/午後Ⅱでの出方

## 7. 頻出テーマ要件

頻出テーマはアプリ内で明示する。

表示場所:

1. ITSMハブ
   - Top 5を表示する。
2. 頻出テーマ画面
   - 全テーマを順位付きで表示する。
3. 知識ノート
   - 各章に頻出度を表示する。
4. 履歴/レポート
   - 将来拡張でテーマ別達成度を表示できる構造にする。

各テーマが持つ情報:

- テーマ名
- 頻出度 `S/A/B`
- 出る区分: 午前Ⅱ / 午後Ⅰ / 午後Ⅱ
- 直近10回分での出題傾向
- まず覚えること
- 午前Ⅱでの問われ方
- 午後Ⅰでの読み方
- 午後Ⅱでの書き方
- インフラ案件での使い方
- 関連する初回対象問題

## 8. 午前Ⅱ要件

午前Ⅱは25問にこだわらず、短い演習を回せるようにする。

機能:

- 公式PDFリンク
- 問番号別の解答入力
- 即時採点
- 1問ずつ演習
- 5問表示
- 追加表示
- 未演習フィルタ
- 直近不正解フィルタ
- 頻出Sテーマフィルタ
- 正答率表示
- 復習メモ

方針:

- 問題文全文はアプリ側に持たせず、公式PDFを参照する答案用紙型とする。
- アプリ側には正答、テーマ、復習メモ、頻出テーマ連携を持つ。
- 合格ラインは60%、アプリ推奨ラインは80%とする。

## 9. 午後Ⅰ要件

午後ⅠはPMアプリと同様、1問50点満点で自己採点する。

機能:

- 令和7年度春期3問一覧
- 問題PDF、解答例PDF、採点講評PDFリンク
- 自分の解答メモ
- 公式解答例/要点の表示
- 独自の採点ポイント
- 採点講評からの注意
- 失点しやすい罠
- 50点満点の自己採点記録
- 問題別履歴

方針:

- 3問中2問の本番形式は将来拡張で扱えるようにするが、初回では1問50点記録を主とする。
- 公式解答例は出典を明示し、学習上必要な範囲で扱う。
- 解説・採点ポイント・失点ポイントは独自作成とする。

## 10. 午後Ⅱ要件

午後Ⅱは最も手厚くする。

各テーマに持たせる情報:

- 問題PDFリンク
- 出題趣旨PDFリンク
- 採点講評PDFリンク
- テーマ要約
- 評価観点
- 骨子例2本程度
- 参考答案2本
- 下書き保存
- 字数カウント
- 自己評価
- 振り返り
- ITSM専用履歴

参考答案方針:

- 各テーマ2本。
- インフラエンジニア案件に寄せる。
- 架空事例とする。
- 実在企業・実在障害には寄せない。
- 「参考答案（論述例の一つ）」と明記し、唯一の正解と誤認させない。
- 2本は似た答案にせず、事例・論点・改善軸を分ける。

自己評価観点:

- 題意適合
- 具体性
- サービスマネジメント活動として書けているか
- 構成
- 振り返り

## 11. ITSM専用履歴要件

既存PM履歴には混ぜず、ITSM内だけで履歴を表示する。

記録対象:

- 午前Ⅱ解答
- 午後Ⅰ採点記録
- 午後Ⅱ練習完了
- 午後Ⅱ参考答案閲覧

記録しないもの:

- 学習時間
- XP
- バッジ
- 既存活動履歴イベント

## 12. 段階リリース

### Phase 1: 設計

- 本要件定義
- 基本設計
- 詳細設計
- 頻出テーマ調査仕様
- 受入条件

### Phase 2: 初回実装

- トップカード
- ITSMハブ
- 頻出テーマ
- 知識ノート
- 午前Ⅱ答案用紙型演習
- 午後Ⅰ自己採点
- 午後Ⅱ骨子・参考答案・自己評価
- ITSM専用履歴

### Phase 3: 初回レビュー

- ユーザ確認
- 表示、内容、演習導線の修正
- 隔離検証

### Phase 4: 5年分拡張

順序:

1. 午前Ⅱ5年分
2. 午後Ⅰ5年分
3. 午後Ⅱ5年分
4. 頻出テーマ再計算
5. 知識ノート増補

## 13. 受入条件

### 13.1 既存PM無影響

- トップページ以外の既存PM画面にITSM導線が増えない。
- 既存検索にITSMデータが出ない。
- 既存学習履歴にITSMイベントが出ない。
- 既存バッジ/XPにITSM結果が反映されない。
- 既存同期にITSMデータが含まれない。
- 既存PM午前Ⅱ、午後Ⅰ、午後Ⅱが従来通り動く。

### 13.2 ITSM機能

- ITSMハブで午前Ⅱ/午後Ⅰ/午後Ⅱの進捗が見える。
- 頻出テーマTop 5が見える。
- 頻出テーマ画面でS/A/Bと試験区分別の出方が分かる。
- 知識ノートで午前Ⅱ/午後Ⅰ/午後Ⅱへの使い方が分かる。
- 午前Ⅱで解答、採点、復習ができる。
- 午後Ⅰで自分の解答、公式解答例、独自ポイント、採点記録が扱える。
- 午後Ⅱで骨子、本文、自己評価、参考答案確認ができる。
- ITSM専用履歴が見える。

### 13.3 品質

- `npm run build` が成功する。
- `npm run validate-data` が既存データの既知警告以外で失敗しない。
- デスクトップとモバイルで主要ページに大きな崩れがない。
- LocalStorageキーに `pmap:sm:*` 以外のITSM保存がない。

## 15. 直近10回の確定定義

2026-06-01時点の直近10回は、IPA公式の年度別過去問題ページでSMのPDF一式が確認できる回を対象にする。

対象:

1. 令和7年度春期
2. 令和6年度春期
3. 令和5年度春期
4. 令和4年度春期
5. 令和3年度春期
6. 令和元年度秋期
7. 平成30年度秋期
8. 平成29年度秋期
9. 平成28年度秋期
10. 平成27年度秋期

令和2年度はSM公開PDFが確認できないため、直近10回の母集団から除外する。

## 16. 初回実装前の必須ゲート

実装へ進む前に、次を満たす。

- `05_source_policy.md` に従い、公式PDFリンクとアプリ内保持範囲を確認する。
- `07_10round_research_matrix.md` のS/A/Bランクを初回データに反映する。
- `06_acceptance_test_plan.md` の既存PM無影響チェックを実装レビュー項目に入れる。
- 既存PMドキュメントを変更しない。
- 追加実装はトップカードとITSM専用ルート・専用ファイルに閉じる。
