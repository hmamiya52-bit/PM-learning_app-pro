# ITサービスマネージャ 公式情報・出典管理ポリシー v1.0

作成日: 2026-06-01

## 1. 目的

ITサービスマネージャ寄り道モードで扱う試験情報、過去問題、解答例、採点講評、出題趣旨を、公式情報に基づいて正確に扱うための方針を定義する。

このモードは短時間合格を狙うために要約・解説・参考答案を多く持つが、問題文や公式資料を無秩序に取り込まない。公式PDFへの導線を残し、アプリ内の本文は学習上必要な独自整理に寄せる。

## 2. 公式ソースの優先順位

優先順位:

1. IPAのITサービスマネージャ試験ページ
2. IPAの過去問題一覧ページ
3. 年度別の問題冊子・解答例・採点講評ページ
4. 各PDF: 午前Ⅱ問題、午前Ⅱ解答例、午後Ⅰ問題、午後Ⅰ解答例、午後Ⅰ採点講評、午後Ⅱ問題、午後Ⅱ出題趣旨、午後Ⅱ採点講評
5. IPAの試験要綱・シラバス

第三者サイト、解説ブログ、予備校資料、生成AIの推測は、公式情報の代替にしない。使う場合も実装データの根拠にはしない。

## 3. 2026-06-01時点の試験前提

IPA公式ページで確認した前提:

- 令和8年度からITサービスマネージャ試験はCBT方式へ移行予定。
- 知識・技能の範囲、出題形式、出題数、試験時間は変更なし。
- 午前Ⅱは25問40分、午後Ⅰは3問中2問90分、午後Ⅱは2問中1問120分。
- アプリ表記はユーザーに馴染みのある `午前Ⅱ` / `午後Ⅰ` / `午後Ⅱ` を主とし、CBT後の名称は補足扱いにする。

## 5. アプリ内に保持してよいもの

保持してよい:

- 年度、期、区分、問番号
- 正答記号
- テーマ分類
- 頻出度
- 公式PDFリンク
- 公式解答例の要点を学習用に整理した短いメモ
- 採点講評から抽出した失点傾向の独自要約
- 午後Ⅰの自己採点ポイント
- 午後Ⅱの骨子、評価観点、インフラ案件の参考答案
- 出題趣旨の独自要約

原則として保持しない:

- 午前Ⅱの問題文全文
- 午後Ⅰの問題文全文
- 午後Ⅱの設問文全文
- 採点講評全文
- 解答例PDFの長い転載
- 図表の転載

## 6. 画面表示ルール

すべての過去問画面で表示する:

- `出典: IPA 令和X年度○期 ITサービスマネージャ試験`
- 公式PDFリンク
- 最終確認日

午前Ⅱ:

- 公式PDFを開いて解く形式を基本にする。
- アプリ内には問番号、テーマ、解答入力、正答、復習メモを置く。
- 問題文全文をアプリ内に持たない。

午後Ⅰ:

- 公式PDFリンクを必ず問題ごとに表示する。
- 公式解答例は、学習に必要な範囲の要点表示に留める。
- 採点ポイント、失点ポイント、読み取り観点は独自作成とする。

午後Ⅱ:

- 公式の出題趣旨を根拠にテーマを要約する。
- 参考答案はすべて独自作成とし、インフラエンジニアの案件にする。
- 参考答案欄には「参考答案の一例であり、唯一の正解ではない」と明記する。

## 7. データ型に必要な出典フィールド

各問題データは、最低限次の出典情報を持つ。

```ts
interface SmSourceLinks {
  ipaPageUrl: string
  questionPdfUrl?: string
  answerPdfUrl?: string
  commentaryPdfUrl?: string
  sourceLabel: string
  checkedAt: string
}
```

`checkedAt` は `YYYY-MM-DD` 形式とする。

## 8. 再確認が必要なタイミング

再確認する:

- 実装着手前
- 直近1年分のユーザーレビュー前
- 5年分拡張前
- IPAが試験制度・シラバス・CBT関連ページを更新したとき
- 年度別過去問題ページに新しいSM回が公開されたとき

再確認時の作業:

1. IPA過去問題一覧で最新年度ページを確認する。
2. SMのPDF一式があるか確認する。
3. `07_10round_research_matrix.md` の10回定義を更新する。
4. 頻出テーマの順位を再集計する。
5. アプリ実装データの `source.checkedAt` を更新する。

## 9. 著作権・引用ガード

IPAの過去問題ページには利用に関する案内があるが、アプリの設計としては過剰転載を避ける。

ガード:

- 問題文全文を持たない。
- 公式PDFを一次参照としてリンクする。
- 公式解答・講評は短い要点または独自要約にする。
- 独自解説と公式資料の境界をUI上で分ける。
- 長文の公式文章をそのままカードやノートに貼らない。

これにより、保守性と学習効率を両立しつつ、将来の公式PDF差し替えにも耐える。
