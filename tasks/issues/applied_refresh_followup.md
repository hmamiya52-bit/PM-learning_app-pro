# 応用情報リフレッシュモード フォローアップ課題

> 作成: Claude（2026-05-26）
> 親レビュー: `tasks/reviews/applied_refresh_claude_review.md`
> 経緯: 改訂レビューで検出した Critical 2 件（C1, C2）は同日コミットで対応済。Major/Minor は本ファイルで保留。

## 🟠 Major（学習体験を阻害するレベル）

### M1. ロードマップが診断結果と無関係に固定表示
**現状**：`AppliedRefresh.tsx` の roadmap 画面で Day1〜Day5 がハードコード。
```
Day 1 診断 / Day 2 開発・要件 / Day 3 PM基礎 / Day 4 基盤・運用 / Day 5 確認
```
セキュリティが弱点と判定されたユーザでも主表示は変わらず、下部の「あなた向け復習候補」chip だけが個別化されている。

**改善案**：
- 診断結果の `weakTopicIds` を起点に Day 別カードの並び替え or 内容差し替え
- または「あなたの弱点は Day X の○○です」という見出しを主表示に格上げ
- 弱点ゼロなら「基礎が整っているため、Day 5 確認テストから始められます」と短縮ロードマップ提示

**規模**：UI 改修（AppliedRefresh.tsx の roadmap セクション、約 50 行）

### M2. 過剰ハイライトで「強調」が機能していない
**現状**：`HIGHLIGHT_GROUPS` が 4 グループ × 計 47 語。特に amber グループ（最初・まず・必要・確認・合意・分析・分類・測定・承認・影響・判断）は普通の動詞・副詞まで全部色付き。実機で 1 段落あたり 5〜10 語がハイライトされ、目がチカチカする。本編ノート規約（==赤＝重要語==／__ネイビー＝構造ラベル__ の 2 色のみ）と乖離。

**改善案**（優先度順）:
1. **行動語グループ（amber）を全廃**
2. rose グループ（警戒語）も「リスクが高い」「漏れ」など強い語のみに絞る
3. 残す indigo（専門用語）と emerald（PM 試験識別子）の 2 色のみに整理
4. または本編ノート規約 `==X==` / `__X__` に寄せる（本文側のテキストに同マーカーを埋め込む）

**規模**：データ修正のみ（`HIGHLIGHT_GROUPS` 配列の絞り込み、約 20 行削減）

### M3. 「復習済み」が自己申告ボタンのみで理解度と無関係
**現状**：`markTopicDone` が `completedTopicIds` 追加のみ。topic.check 問題に正解していなくても押せる。stats の `${completedCount}/8` が学習達成度を表していない。

**改善案**：
- 条件付け：「topic.check に正解 + 押下」または「topic.check 自体の正誤で自動完了」
- 後者の方が UI 操作が減ってシンプル
- 既存の自己申告フローも残すなら、検定ありの "認定完了" と検定なしの "復習済み" を区別

**規模**：UI ロジック改修（AppliedRefresh.tsx の `markTopicDone` 周辺、約 20 行）

### M4. モバイル幅で「弱点復習」がメニュー縦並びファースト
**現状**：`lg:grid-cols-[240px_1fr]` で lg 未満は縦積み。弱点復習タブを開くとまず 8 個のトピックボタンが縦に並び、本文を読むには長距離スクロール。

**改善案**：
- モバイルではメニューを折り畳み（accordion / drawer）にして、デフォルトで最初の topic 本文を即表示
- または弱点トピックが抽出済なら自動的にその topic を選択して開く

**規模**：UI 改修（AppliedRefresh.tsx の refresh セクション + 状態初期化、約 40 行）

## 🟡 Minor（仕上げレベル）

### m1. リセットボタンに確認ダイアログなし
- 1 クリックで進捗を全消去
- 改善案：`if (!window.confirm('リセットしますか？...')) return` を `resetMode` 冒頭に追加

### m2. デフォルト画面が常に diagnostic
- 2 回目以降の訪問者にも診断画面が表示される
- 改善案：LocalStorage に進捗があれば `useState<Screen>(() => loadState().diagnosticAnswers ? 'refresh' : 'diagnostic')` に変更

### m3. フラッシュカードが各トピック 2 枚のみ
- 応用情報のリフレッシュとしては薄い
- 改善案：各トピック 4〜6 枚に拡充。RTO/RPO や請負/準委任/派遣のような重要対比は 1 枚に集約せず別カードへ

### m4. 確認テスト合格時に PM 本編への直接リンクがない
- 「8/10 以上で午前II・午後Iへ進める」とメッセージは出るが導線がない
- 改善案：`finalScore >= 8` 時に `<Link to="/morning">午前II演習へ</Link>` `<Link to="/afternoon">午後I演習へ</Link>` を表示

### m5. 自己申告と客観指標の二重管理が画面に出ていない
- `topicCheckAnswers`（客観指標）が画面の stats に反映されていない
- 改善案：「復習: ${completedCount}/8」の隣に「内、確認問題正解: ${topicCheckCorrectCount}/8」を併記
