## 🟠 Major（学習体験を阻害するレベル）

### M1. ロードマップが診断結果と無関係に固定表示 【✅ 2026-05-26 修正済】

**修正前の問題**：roadmap 画面が Day1〜Day5 ハードコード。診断結果に関わらず固定。

**実施した修正**（src/pages/AppliedRefresh.tsx の roadmap セクション 動的化）:
- 診断状況 × 弱点数で 3 分岐:
  - 診断未完了: Day1 で診断を促す + 固定 Day2-5 サマリ
  - 診断完了 + 弱点なし: 短縮ルート（Day1 完了 + Day2 確認テスト直行）
  - 診断完了 + 弱点あり: Day1 完了 + 弱点トピックを Day2-N で並べる + 最終 Day で確認テスト
- 各 Day カードに **action ボタン**（このテーマを復習 / 確認テストへ / 弱点復習へ）を埋め込み、ロードマップから直接画面遷移
- 各 Day カードのトーン色分け（pending=灰 / done=緑 / highlight=amber / final=brand）
- 見出しも個別化「あなたの弱点 N テーマを順番に復習する個別ロードマップです」
- 弱点が 4 件超のときは「+ 他 N テーマ」枠で誘導
- 完了済みトピックは Day カード自体が緑表示 + 「再復習する」ボタンに変化

**結果**：
- ロードマップが完全に診断結果連動の個別表示へ
- 実機で「弱点 3 件 / 要件定義復習完了」状態を視認、Day 2 (要件定義) が緑＋「✓ / 復習完了済」表示、Day 3-4 が amber、Day 5 が brand で連動

### M2. 過剰ハイライトで「強調」が機能していない 【✅ 2026-05-26 修正済】

**修正前の問題**：`HIGHLIGHT_GROUPS` が 4 グループ × 計 47 語。特に amber グループ（最初・まず・必要・確認・合意・分析・分類・測定・承認・影響・判断）は普通の動詞・副詞まで全部色付き。実機で 1 段落あたり 5〜10 語がハイライトされ、目がチカチカ。

**実施した修正**（src/pages/AppliedRefresh.tsx の HIGHLIGHT_GROUPS 改修）:
- 行動語グループ（amber）を全廃: 11 語削除
- 警戒語グループ（rose）を全廃: 11 語削除
- indigo 専門用語からも普通語を除外: 要求・要件・仕様・品質・レビュー・テスト・リスク・ベースライン（8 語）削除
- emerald（PM 試験識別子）と indigo（厳選した固有・技術用語）の 2 色のみに整理
- 47 語 → 22 語、4 色 → 2 色に絞り込み

**結果**：
- 1 段落あたりハイライト語が 5〜10 → 1〜2 に減少
- 本編ノート規約（赤＝重要語／ネイビー＝構造ラベルの 2 色のみ）の節度ある強調設計に準拠
- 視覚情報の優先順位が明確に（PM 試験識別子・固有用語のみが目立つ）
- 実機（mobile）の弱点復習画面で視認確認、過剰色付け解消を確認

### M3. 「復習済み」が自己申告ボタンのみで理解度と無関係 【✅ 2026-05-26 修正済】

**修正前の問題**：`markTopicDone` が `completedTopicIds` 追加のみ。topic.check 正誤と無関係に「復習済み」を押せる。

**実施した修正**（src/pages/AppliedRefresh.tsx の answerTopicCheck / UI ボタン部分）:
- `answerTopicCheck` で正解時に自動的に `completedTopicIds` へ topicId を追加
- 自己申告ボタン「このテーマを復習済みにする」を削除
- 代わりに状態表示へ:
  - 未挑戦: 「確認問題に挑戦して達成度を更新」灰色チップ
  - 正解: 「✓ このテーマは復習完了（確認問題に正解）」緑バッジ
  - 不正解: 「確認問題に再挑戦してください」amber チップ
- `markTopicDone` 関数自体を削除（呼び出し箇所なし）

**結果**：
- stats の `completedCount/8` が客観指標（topic.check 正解数）と一致
- ユーザは挑戦せずに完了マークできない → 学習達成度が正確に反映
- 実機で確認問題 B 選択 → 即座に状態表示が「✓ 復習完了」へ、かつロードマップ Day カードも自動で完了表示に連動更新を視認

### M4. モバイル幅で「弱点復習」がメニュー縦並びファースト 【✅ 2026-05-26 修正済】

**修正前の問題**：lg 未満ではトピック一覧が縦積みで先頭表示、本文まで長距離スクロール。

**実施した修正**（src/pages/AppliedRefresh.tsx の screen='refresh' セクション + 初期化ロジック）:
- モバイル幅（lg 未満）ではトピック一覧を `<details>` で **折畳み式**に変更
  - 折畳みヘッダに「テーマを切り替えるにはタップ（現在: 要件定義）」表示
  - 開閉した時のトピック選択でモバイルでは即座に本文へ smooth scroll
  - 折畳み内は 2 列グリッドでコンパクト配置
- デスクトップ（lg 以上）では従来通り左サイドバー固定
- **初期状態の個別化**:
  - LocalStorage に進捗があれば 2 回目以降は `diagnostic` 画面ではなく `refresh` 画面で起動
  - 初期 activeTopic は弱点 1 件目（弱点なしならトピック先頭）
  - 「8 個のトピックを縦に見せられる」問題を根本解決

**結果**：
- モバイルで `/applied-refresh` を開いた瞬間に「弱点復習」タブ + 弱点 1 件目（要件定義）の本文が即表示
- トピック切り替えは折畳みアコーディオンから（占有スペース最小化）
- 実機（375×812）で初回〜2 回目訪問の動線を視認確認

## 🟡 Minor（仕上げレベル）

### m1. リセットボタンに確認ダイアログなし
- 1 クリックで進捗を全消去
- 改善案：`if (!window.confirm('リセットしますか？...')) return` を `resetMode` 冒頭に追加

### m2. デフォルト画面が常に diagnostic
- 2 回目以降の訪問者にも診断画面が表示される
- 改善案：LocalStorage に進捗があれば `useState<Screen>(() => loadState().diagnosticAnswers ? 'refresh' : 'diagnostic')` に変更

### m3. フラッシュカードが各トピック 2 枚のみ
- 応用情報のリフレッシュとしては薄い
- 改善案：各トピック 4〜6 枚に拡充。RTO/RPO や請負/準委任/派遣のような重要対比は 1 枚に集約せず別カードへ

### m4. 確認テスト合格時に PM 本編への直接リンクがない
- 「8/10 以上で午前II・午後Iへ進める」とメッセージは出るが導線がない
- 改善案：`finalScore >= 8` 時に `<Link to="/morning">午前II演習へ</Link>` `<Link to="/afternoon">午後I演習へ</Link>` を表示

### m5. 自己申告と客観指標の二重管理が画面に出ていない
- `topicCheckAnswers`（客観指標）が画面の stats に反映されていない
- 改善案：「復習: ${completedCount}/8」の隣に「内、確認問題正解: ${topicCheckCorrectCount}/8」を併記
