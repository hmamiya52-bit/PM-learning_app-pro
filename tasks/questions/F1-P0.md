## Claude からの回答（2026-05-14）

- **A1: YES — `src/data/questions/index.ts` の `protocol-review` 関連2行を削除してください。**

  具体的に削除する行:
  - line 42: `import { protocolReviewQuestions } from './protocol-review'`
  - line 71: `...protocolReviewQuestions,`

  これは F1-P0 指示書 §7 補足 Case 1 と一致する想定通りの対処です。`src/data/questions/protocol-review.ts` は Step 2 で Claude が削除済みのため、import 行を残しておくとビルドが通りません。

- **A2: YES — `categories.ts` や `NoteDetail.tsx` の `protocol-review` ID 参照は F1-P0 では触らないでください。**

  理由: これらは **配列の文字列リテラル**（例: `'protocol-review'`）として登録されているだけで、import 文ではないためファイル削除によるビルドエラーには直結しません。

  19 個の NW固有カテゴリ全部（`layer1-3`, `routing`, `dns`, `dhcp`, `http`, `tcp-ip`, `mail`, `ssl-tls`, `ipsec`, `firewall`, `ids-ips`, `wireless`, `vrrp`, `voip`, `wan`, `monitoring`, `security-attack`, `ldap-auth`, `ipv6`, `iot`, `security`, `load-balancer`, `sdn`, `proxy`, `protocol-review` 等）は **F1-P1（カテゴリ・サイドバー）で本格的に PM 12カテゴリへ置き換え** ます。F1-P0 の最小修正は index.ts の 2行削除のみで OK。

  Quiz.tsx (NoteDetail.tsx から NOTE_CATEGORY_IDS を import している) もそのまま動作するはずです（カテゴリ名がNW固有のままなだけで、ビルドは通る）。
