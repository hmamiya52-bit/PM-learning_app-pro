### 結論
v1.0.0 段階では「独立カテゴリ新設」は **闇雲に量を増やす**結果になり、学習効率を下げる。各カテゴリ末尾に「位置づけ」注記を加えるのみとし、試験要綱反映後に拡充する方針。

## 2. PMBOK 第8版の概要

### 6 原則（第7版の 12 原則を統合）
1. **全体的な視点を持つこと**（Adopt a Holistic View）
2. **価値に焦点を当てること**（Focus on Value）
3. **品質をプロセスと成果物に組み込むこと**（Build Quality into Processes and Deliverables）
4. **責任あるリーダーであること**（Be an Accountable Leader）
5. **持続可能性をすべてのプロジェクト領域に統合すること**（Integrate Sustainability）
6. **自律的な文化を築くこと**（Build an Empowered Culture）

### 7 パフォーマンス領域（第7版の 8 領域を統合・改名）
1. **ガバナンス**
2. **ステークホルダー**
3. **スコープ**（品質含む）
4. **資源**
5. **スケジュール**
6. **リスク**
7. **ファイナンス**

→ 興味深いことに、第8版は第7版の原則指向を保ちつつ第6版の知識エリア体系（スコープ・スケジュール・リスク等）に回帰。

## 3. 12 カテゴリ × 第8版 統合マッピング

| 既存カテゴリ | 第8版での主な対応 | 注記の焦点 |
|---|---|---|
| **stakeholder** | 領域「ステークホルダー」継続。原則「責任あるリーダー」追加 | 名称変更なし、責任あるリーダーシップが原則として明文化 |
| **team** | 領域「資源」へ統合。原則「自律的文化」追加 | 第7版「チーム」領域は第8版で「資源」に統合、自律的文化が原則化 |
| **development-approach** | 領域として独立廃止、テーラリング+原則「価値焦点」に分散 | 第8版 7 領域に「開発アプローチ」なし、価値原則と修整の文脈に統合 |
| **planning** | 領域「スコープ」「スケジュール」「ファイナンス」に分割 | 第6版回帰、計画は領域別に細分化、用語は第6版方向 |
| **project-work** | 領域として独立廃止、全領域に分散 | 第7版「プロジェクト作業」は第8版で領域横断要素として再整理 |
| **delivery** | 領域として独立廃止、原則「価値焦点」「品質組込」に統合 | 第8版「品質」は「スコープ」領域に内包、価値は原則格上げ |
| **measurement** | 領域「ファイナンス」へ統合 | 第7版「測定」は第8版「ファイナンス」に統合（EVM 等） |
| **uncertainty** | 領域「リスク」に名称変更・統合 | 「不確かさ」→「リスク」へ第6版方向の名称回帰 |
| **integration** | 領域として独立廃止、原則「全体的な視点」に統合 | 第8版で「全体観」が原則として明文化、統合は領域横断概念に |
| **governance** | 領域「ガバナンス」継続。原則「持続可能性」追加 | 持続可能性が新規原則として明文化、ガバナンスに長期視点 |
| **tailoring-models** | 原則「全体的な視点」「価値焦点」と関連 | テーラリング自体は領域から原則・全体方針に格上げ |
| **service-management** | 領域「ファイナンス」+ 原則「持続可能性」 | サービス運用の長期持続性視点が原則化 |
