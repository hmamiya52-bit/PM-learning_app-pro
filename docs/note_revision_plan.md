## 2. 方針A: 赤字リストラ（全カテゴリ）

**基準**: 「午前Ⅱの正答として選ぶ語」または「午後Ⅰの解答に書く語」だけを赤字にする。
**目安**: 3〜6個/セクション（stakeholder 水準）。超過する場合はコメントで理由を残す。

### 降格ルール
| ルール | 対象 | 降格先 |
|---|---|---|
| A-1 | 汎用ITTO語: 専門家の判断／会議／データ分析／データ収集／データ表現／意思決定／対人スキル／プロジェクト文書の更新／変更要求（アウトプット列挙文脈）／PMIS／コミュニケーション・スキル／代替案分析 | 平文 |
| A-2 | 行頭ラベル語: 目的／プロセス／KPI／成果物／技法／主要アウトプット／定義 等 | `__ネイビー__` |
| A-3 | 説明断片・副詞・形容: 「迅速に」「計画的」「最小化」「スコープが明確」型 | 平文 |
| A-4 | 番号+用語の結合: `==1. 回避==` | `__1.__ ==回避==` に分離 |
| A-5 | 一般語: 低／中／高／複雑／プロジェクト／チーム／コスト／計画 等（固有概念の構成要素でない場合） | 平文 |
| A-6 | プロセス宣言行の「N.N プロセス名」（例: `==5.4 WBS作成==`）。節番号は暗記対象でなく、セクション見出しとも重複するため | 平文（プロセス群所属も平文） |

> **A-6 は 2026-07-11 パイロットレビューでの顧客指摘**（「節の番号とか赤字にする必要ない」）。
> 疑わしい赤字は「試験の解答用紙にその語を書くか？」で判定し、迷ったら降格する。

### 残す赤字（変更しない）
- 固有用語（サリエンスモデル、クラッシング、TCPI、KEDB…）
- 式・数値（EV−AC、BAC/CPI、99.73%…）
- 分類の正式名称（回避／転嫁／軽減／受容／エスカレーション…）
- 対比キー（MTTR vs MTBF…）
- 判定基準値（CPI > 1、TF = 0…）

## 3. 方針B: 「午後Ⅰの定石」セクション新設（全カテゴリ）

- 各カテゴリ末尾、「過去問頻出論点（午前Ⅱ）」の後に **「N. 午後Ⅰの定石」** を新設
- ソース: `afternoonExplanations.ts` の keyKnowledge 438項目をカテゴリへマッピングし、重複統合（推定100〜150定石 → 10〜15項目/カテゴリ）
- 形式: `__定石名__: 一行説明（赤字は解答キーワードのみ）`、出題実績（例: R6問2）は navyItems へ
- 例: 「__リスク転嫁の限界__: 請負での再委託でも==スケジュールのリスク==は実質移せない。予兆は隠さず共同管理が定石」
- 「設問の限定語の読み方」等の解答技術系は、パイロット時は planning の定石セクション内に含め、全展開後に独立配置の要否を判断

## 4. 方針C: ITTO羅列の圧縮＋情報自体の削減

- プロセス定義セクション（「N.N 〜」型見出し）の入力／技法／出力列挙を **試験で問われる代表3〜5個** に圧縮
- 残す項目の判定: `officialMorningQuestions.ts` に出題実績がある項目は残す
- 圧縮後も赤字は解答実績のある成果物名等のみ（方針Aと同基準）
- セクション自体は削除しない（理解度記録が `categoryId:sectionIndex` で保存されており、削除すると既存記録がずれるため）

### C-2: 行レベルの情報削減（2026-07-11 顧客指示「削っていい情報は削る」）

判定基準: **「この行が無いと午前Ⅱ／午後Ⅰで失点するか？」** No なら削る。

| 削る対象 | 例 |
|---|---|
| 抽象論・お題目（PMBOK7 の「成果」列挙等） | 「プロジェクト目標に対する進捗の組織化・整合化・連携」 |
| 見出しと重複するプロセス宣言行（**説明文と統合した形も含め全廃**、2026-07-11 顧客FB） | 見出し「5. 5.1 スコープマネジメント計画」があるのに本文1行目が「5.1 スコープマネジメント計画（Plan Scope Management）: …」。プロセス名・英語名を落とし「…を文書化するプロセス」のような説明文だけにする。1セクション複数プロセスの場合のみ `__妥当性確認__:` 形式の短いネイビーラベルで区別 |
| 名前から自明なアウトプット列挙 | 「アクティビティ・リスト／アクティビティ属性／マイルストーン・リスト」 |
| 他セクションと重複する比較・再掲 | §21 の4技法比較（§15 と同内容）、§30 の成果列挙（§1 と同内容） |
| マイナー用語 | コードオブアカウント、プランニング・オニオン、トライアンギュレーション |
| navyItems と重複する「試験頻出:」行 | 本文末尾の「試験頻出: NPV計算問題」 |
| **出典表記**（2026-07-11 顧客指示「出典とかいらない」） | navyItems の「出典: PMBOK第6版 §5.1」等。出典のみの navyItems は行ごと削除、有用情報と混在する場合は出典部分のみ削る。出題実績（「R6問2」「午前Ⅱ出題実績あり」）は出典ではなく学習情報なので残す |
| exam_tips の頻出論点セクション（§32）との重複 | exam_tips は §32 に無い最重要 5 個前後に絞る |

- 複数行の列挙は「1行への畳み込み」を優先（親見出し行＋子リスト → 1行の「／」区切り）
- プロセス群所属は原則平文で削除可。ただし**監視・コントロール群のプロセス**（5.5/5.6/6.6/7.4 等）は計画群との判別がひっかけになるため「監視・コントロール群」表記を残す
